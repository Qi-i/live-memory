# 账号、安全删除与轻量备份设计

## 目标

Live Memory 账号负责身份资料、密码找回和小体积文字备份。用户自己的 Supabase 是可选的完整同步空间，可决定是否上传图片。演出图片默认保留在本机，完整 JSON 备份用于迁移和长期保存。

## 数据流

- 账号资料：昵称、用户名、压缩头像、找回邮箱保存到账号云端。
- 文字备份：演出字段写入 `echo_text_backups`，媒体数组在上传前移除。
- 完整同步：记录写入用户自己的 `echo_passkey_records`；开启图片同步时再上传 Storage。
- 自动备份：记录变更后延迟执行，且遵守用户选择的最短备份间隔。
- 删除：第一次删除只写入 `deletedAt` 并进入回收站；恢复会清除该字段；永久删除需要再次确认。

## 约束

- 用户名为 4-32 位英文字母或数字，统一转为小写。
- 密码至少 8 位，不限制字符种类。
- 找回邮箱只属于 Live Memory 账号，不写入个人 Supabase 设置。
- 头像只接受图片，原文件上限 10MB，保存前缩放到 320px。
- 账号表使用 `user_id` 与 RLS 隔离用户；个人完整同步使用同步钥匙与 RLS 隔离档案。

## 恢复策略

文字备份恢复时，以 `updatedAt` 较新的版本为准；同 ID 的本地图片始终保留。回收站记录也参与备份，避免另一台设备把已删除记录重新带回。
