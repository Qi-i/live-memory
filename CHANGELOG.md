# 更新记录

## 2.1.6 - 2026-06-20

### 修复

- 已登录 Live Memory 账号时，连接个人 Supabase 不再要求再次输入个人云端密码。
- 个人 Supabase 缺少同步表时，直接提示运行 `005_passkey_cloud_sync.sql`，不再只显示笼统的操作失败。
- 个人云端图片同步保留本地图片缓存，避免 Supabase 临时图片链接过期后变成黑图。
- 设置页去掉重复顶部区块，并统一主要内容宽度。
- 票夹视图强制左右栏分离，避免文字压到海报上。

### 文档

- 将普通用户 Supabase 教程改为只运行个人云端 SQL，并区分公开站点账号登录和无账号服务自部署。

## 2.1.5 - 2026-06-20

### 修复

- 首次使用选择“完整云同步”时，不再自动触发 Live Memory 账号邮件登录。
- 个人云端已连接后隐藏密码输入，只在更换项目、用户名或主动重新连接时再输入。
- 将账号邮件频率限制错误改为中文提示，避免和个人 Supabase 连接混淆。

## 2.1.4 - 2026-06-20

### 调整

- 整理 README 使用入口，明确公开地址、账号文字备份、个人 Supabase、本地运行和自部署的区别。
- 优化 Supabase 配置指南，分开普通用户和站点维护者的操作路径。
- 精简设置页账号、保存和个人云端提示，减少工程化说明。
- 补充 package 关键词，便于 GitHub 识别项目用途。

## 2.1.3 - 2026-06-20

### 修复

- 未登录账号时，账号文字备份不再显示 `Auth session missing!`，改为清晰的中文提示。
- 个人云端上传继续使用同步钥匙模式，避免账号备份状态干扰完整云同步。

## 2.1.2 - 2026-06-20

### 修复

- 个人 Supabase 完整同步改为同步钥匙模式，不再创建个人 Supabase 邮件账号。
- 连接个人云端后才允许上传和恢复，避免 `Auth session missing` 这类底层提示。
- 新增 `005_passkey_cloud_sync.sql`，为免邮件同步建立数据表、RLS 和图片空间访问规则。
- 账号密码与个人云端密码分开，找回邮箱只用于 Live Memory 密码找回。

### 文档

- 更新 Supabase 设置教程、数据同步说明和 GitHub Pages 发布说明。
- 将个人完整同步表名更新为 `echo_passkey_records` 与 `echo_passkey_media_assets`。

## 2.1.1 - 2026-06-20

### 调整

- 首次进入同时支持登录已有账号和创建新账号。
- 已登录时更新账号资料不会重复创建账号。
- 账号服务未连接时，页面改为明确的当前设备保存流程，不再显示不可用的云备份操作。
- GitHub Pages 部署兼容旧版 Supabase 变量名。
- 恢复 Vite 标准依赖预打包，修复本地开发模式的 React 空白页面。
- 提高 Supabase 设置区文本链接在浅色背景上的对比度。
- 增加将现有 25 条私人记录从本地页面迁入个人 Supabase 的完整步骤。

## 2.1.0 - 2026-06-20

### 新增

- 头像图片选择、压缩和账号展示。
- 账号文字备份、自动备份和备份间隔设置。
- 个人 Supabase 图片同步开关。
- 删除二次确认、回收站恢复和永久删除确认。
- 账号密码找回与密码更新入口。
- 文字 JSON 备份导出。

### 调整

- 用户名限定为 4-32 位英文字母或数字。
- 密码最低 8 位，不限制字符种类。
- 账号邮箱仅用于 Live Memory 账号确认和密码找回。
- Live Memory 账号云端与用户个人 Supabase 数据空间分离。

### 数据库

- 新增 `004_account_backup_and_validation.sql`。
- 新增 `echo_text_backups` 私人文字备份表。
- 增加用户名格式约束和索引。
