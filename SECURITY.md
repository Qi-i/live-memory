# 安全策略

## 支持范围

安全修复优先覆盖 `main` 上的最新版本。旧部署请先升级后复现。

## 私下报告

请使用 GitHub 仓库的 Private vulnerability reporting（若已启用）报告漏洞，不要在公开 Issue 中附上可利用细节、Supabase key、个人票根或备份文件。

报告请包含：影响范围、复现步骤、受影响文件或页面，以及建议修复方向。维护者确认后会尽快给出处理状态。

## 使用者注意事项

- anon/publishable key 可用于前端，但必须配合正确 RLS。
- `service_role`、数据库密码、对象存储 Secret Key 不得进入前端或 Git。
- 定期导出本地备份，并妥善保存。
- 公开部署前先用普通用户账号验证无法读取其他用户的数据。
- 票根、座位图和现场照片可能包含姓名、二维码、订单号或精确座位，分享截图前应检查并打码。
